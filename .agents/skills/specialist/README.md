# Specialist Skills

Use this category for repeatable expert passes that are not part of the core process loop, such as
code simplification, architecture audit, naming review, testing discipline, diagnosis, product
strategy, UX direction, creative direction, technical architecture, validation strategy, or refactor
support.

Keep each skill self-contained under `<skill-name>/SKILL.md`.

Current specialist skills:

- `tdd`: behavior-first red/green/refactor through the highest useful public seam.
- `diagnose-bug`: tight red-capable bug feedback loop before implementation.
- `product-strategy`: advisory product, audience, value, market, and differentiation lens.
- `technical-architecture`: advisory platform, ownership, build/runtime, and architecture lens.
- `validation-strategy`: advisory acceptance evidence, feedback-loop, and quality strategy lens.
- `ux-direction`: conditional UX, interaction, information architecture, and accessibility lens.
- `creative-direction`: conditional creative, brand, game-design, tone, and game-feel lens.
