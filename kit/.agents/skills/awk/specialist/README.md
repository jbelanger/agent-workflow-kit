# Specialist Skills

Use this category only for specialist skills with concrete procedure that is not already covered by
the process loop. Product, UX, creative, architecture, and validation lenses live inside
`process/discover-vision`; do not split them into separate "think like X" files.

Keep each skill self-contained under `<skill-name>/SKILL.md`.

Current specialist skills:

- `tdd`: behavior-first red/green/refactor through the highest useful public seam.
- `diagnose-bug`: tight red-capable bug feedback loop before implementation.
